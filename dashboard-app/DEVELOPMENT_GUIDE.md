# 欧洲汽车市场洞察 - 核心开发规范指南

本文档旨在记录本项目的核心业务逻辑、数据处理机制及 UI 交互规范。后续进行二次开发或组件新增时，请严格遵循以下准则。

## 1. 核心数据回溯与修正算法 (The CY/LY Correction Logic)
这是本项目最核心的业务逻辑。由于官方数据具有“次年修正”的特性，我们在所有涉及历史查询的图表中，**必须**遵循以下查询与聚合规范：
* **查询范围外延**：查询特定年份（如 2024 年）的数据时，SQL 查询条件必须强制向后延伸一年（拉取至 2025 年）。
* **CY/LY 取舍机制**：按“年-月”进行极细颗粒度遍历。
  - 如果数据库存在 `次年当月` 的数据，**强制采用次年的 `ly` (Last Year) 字段**（包含 `monthly_units_ly` 和 `monthly_share_ly`）作为最终修正数据。
  - 如果 `次年当月` 无数据（或尚未发布），则采用 `当年当月` 的 `cy` (Current Year) 字段。
* **YTD (年度累计) 计算**：必须基于上述“CY/LY 修正后的月度时间线数组”进行动态累加，严格截断至当前选中的 `selectedMonth`，以保证同比 (YoY) 逻辑的公平性。

## 2. 品牌与集团解耦算法 (Standalone Brand Logic)
在 `manufacturer_regional_registrations` 表中，集团与品牌的区分规则如下：
* **集团数据**：`parent_group === brand`
* **品牌数据**：需通过集合 (Set) 动态判断。如果一个集团下**有且仅有它自己**一个品牌（如 Tesla, Mazda，即 `parent_group === brand` 且该 group 集合长度为 1），它也必须被强行归入“品牌排行榜”中。

## 3. UI/UX 视觉与交互规范
* **主色调统一**：全面使用 Tailwind 的科技蓝 (`blue-600` / `#3b82f6`)。禁止在基础组件中混用高饱和度的粉色、紫色，保持数据大屏的专业、克制。
* **ECharts 渲染优化**：
  - **动态标签 (Inside Placement)**：当条形图数据占比超过最大值的 30%~35% 时，标签应变为白色 (`#ffffff`) 并嵌入柱子内部 (`insideRight`)；过短则为灰色放在外部。这能极大节省网格空间。
  - **背景高亮无损截断 (MarkArea 替代)**：涉及 DataZoom 左右滑动时，严禁使用原生 `markArea`（容易产生边缘裁切）。必须通过 `type: 'custom'` 自定义渲染一个 `z: -1` 的底层矩形来实现当月的高亮光斑。
* **组件彻底解耦**：
  - 图表卡片与弹窗必须分离。
  - 弹窗外壳 (`BaseModal.vue`) 与业务视图 (`xxxDetailView.vue`) 必须解耦。通过 `<slot>` 传递内容。
* **高级拟物态交互**：抛弃浏览器原生 `<select>`，使用自定义的 `CustomSelect.vue`；右侧滚动侧边栏需通过精密的 `top-[104px]` 实现带呼吸感的 `position: sticky` 吸顶效果。

## 4. 突破分页限制
使用 Supabase 时，如果导出全量 Excel 报表，必须使用 `while(true)` 配合 `.range(from, from + 999)` 的轮询引擎，突破底层 1000 行限制。
